# 创建 PR

## 概述

创建结构良好的拉取请求，包含适当的描述、标签和审查者。

## 步骤

1. **准备分支**

   - 确保所有更改已提交
   - 推送分支到远程仓库
   - 验证分支与主分支保持同步

2. **编写 PR 描述**

   - 清晰总结更改内容
   - 包含上下文和动机
   - 列出任何破坏性更改
   - 如有 UI 更改请添加截图

3. **设置 PR**
   - 创建具有描述性标题的 PR
   - 添加适当的标签
   - 分配审查者
   - 关联相关问题

## PR 模板

- [ ] 功能 A
- [ ] 错误修复 B
- [ ] 单元测试通过
- [ ] 手动测试完成
